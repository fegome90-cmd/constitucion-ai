# reviewctl Agent Guide — constitucion-ai

## Propósito

Guía de operación de `reviewctl` para el repositorio `constitucion-ai`.
Este documento es leído por el agente de review antes de iniciar cualquier run.

| Atributo | Valor |
|---|---|

| Review type | `general` |
| Static gate principal | `markdownlint-cli2` |
| Frontend | Deshabilitado |
| Biome | Deshabilitado |
| Ruff | Deshabilitado |

## Prerequisitos

Antes de cualquier `run`, verificar que los siguientes elementos estén presentes:

```bash
# Herramientas
command -v bun                  # runtime reviewctl
command -v jq                   # parsing JSON
command -v markdownlint-cli2    # static gate principal

# CLI
REVIEW_CLI="/Users/felipe_gonzalez/Developer/branch-review/mini-services/reviewctl/src/index.ts"
[ -f "$REVIEW_CLI" ] && echo "REVIEW_CLI_OK"

# Token (cargar desde .env si no está en el shell)
[ -n "$REVIEW_API_TOKEN" ] && echo "TOKEN_OK" || export $(grep -v '^#' .env | xargs)

# Directorio de output
mkdir -p docs/reviews
```

**No iniciar `run` hasta que todos los prerequisitos estén OK.**

## Flujo canónico

```bash
export $(grep -v '^#' .env | xargs)
REVIEW_CLI="/Users/felipe_gonzalez/Developer/branch-review/mini-services/reviewctl/src/index.ts"

# 1. Init
bun "$REVIEW_CLI" init --create

# 2. Explore
bun "$REVIEW_CLI" explore context
bun "$REVIEW_CLI" explore diff

# 3. Plan
bun "$REVIEW_CLI" plan --type general || bun "$REVIEW_CLI" plan

# 4. Static gate — markdownlint
bash scripts/reviewctl/lint_markdownlint_changed.sh > /tmp/markdownlint-output.md 2>&1
bun "$REVIEW_CLI" ingest --static markdownlint --input /tmp/markdownlint-output.md

# 5. Run
bun "$REVIEW_CLI" run

# 6. Verdict
bun "$REVIEW_CLI" verdict
```

---

## Static Gate: markdownlint

### Ejecutar

```bash
bash scripts/reviewctl/lint_markdownlint_changed.sh > /tmp/markdownlint-output.md 2>&1
```

El script:

- Detecta archivos `.md` cambiados respecto a `main...HEAD`
- Si no hay cambios `.md`, corre un check mínimo sobre el primer `.md` del repo
- Exit 0 = PASS, non-zero = FAIL
- La salida siempre incluye `All checks passed!` en PASS para que el parser sea conclusivo

### Ingestar

```bash
bun "$REVIEW_CLI" ingest --static markdownlint --input /tmp/markdownlint-output.md
```

### Verificar que no quedó UNKNOWN

```bash
RUN_ID=$(ls -1dt _ctx/review_runs/run_* | head -n 1 | xargs basename)
jq -r '.statics.markdownlint.status // "MISSING"' "_ctx/review_runs/${RUN_ID}/final.json"
```

Si retorna `UNKNOWN`, regenerar la salida y reingestar antes de llamar a `verdict`.

## Verificación de plan y agentes

```bash
RUN_ID=$(ls -1dt _ctx/review_runs/run_* | head -n 1 | xargs basename)

# Verificar plan.json
test -f "_ctx/review_runs/${RUN_ID}/plan.json" && echo "PLAN_OK" || echo "PLAN_MISSING"
jq -r '.plan_status // "MISSING"'  "_ctx/review_runs/${RUN_ID}/plan.json"
jq -r '.required_agents[]?'        "_ctx/review_runs/${RUN_ID}/plan.json"
```

Reglas:

- `FOUND` y `AUTOGENERATED` son estados válidos de plan.
- `MISSING` / `AMBIGUOUS` son no-bloqueantes. Continuar con `run`.
- Si `run` rechaza por plan, reintentar con `bun "$REVIEW_CLI" run --no-plan` (documentar motivo).

## Criterios de Done

El run se considera completo cuando:

- Verdict es `PASS` o `FAIL` (nunca `INCOMPLETE`)
- Static `markdownlint` no está en estado `UNKNOWN`
- Los valores del summary local coinciden con `final.json`

```bash
RUN_ID=$(ls -1dt _ctx/review_runs/run_* | head -n 1 | xargs basename)
cat "_ctx/review_runs/${RUN_ID}/final.json"
```

## Output y artefactos

| Artefacto | Path |
|---|---|
| Fuente de verdad (JSON) | `_ctx/review_runs/<run-id>/final.json` |
| Resumen legible | `_ctx/review_runs/<run-id>/final.md` |
| Summary de sesión | `docs/reviews/<run-id>-summary.md` |
| Debt log acumulativo | `docs/reviews/lint-debt-backlog.md` |
| Static markdownlint raw | `/tmp/markdownlint-output.md` |

> `_ctx/` está en `.gitignore` — los artifacts no se trackean.
> `docs/reviews/` está en `.gitignore` — los summaries son locales.

## Debt Log

Cuando el scope es `changed-files` (modo default), actualizar el debt log con hallazgos
de archivos **no incluidos** en el diff del PR:

```bash
# Crear o actualizar
docs/reviews/lint-debt-backlog.md
docs/reviews/<run-id>-debt-log.md
```

Campos requeridos: herramienta, hallazgos principales, paths excluidos, motivo de exclusión, fecha, run-id.

## Manejo de fallos

| Situación | Acción |
|---|---|
| Prerequisito faltante antes de `run` | Detener y reportar el prerequisito exacto |
| `plan.json` ausente tras `plan` | Reintentar `plan` una vez; si falla, detener |
| Static `markdownlint` → `UNKNOWN` | Regenerar output, reingestar, correr `verdict` de nuevo |
| `run` retorna `INCOMPLETE` | Inspeccionar agentes requeridos, ingestar explícitamente, correr `verdict` |
| Agentes requeridos ausentes en `final.json` | Detener y reportar cuáles no completaron |
